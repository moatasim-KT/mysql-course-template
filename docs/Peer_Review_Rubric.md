# Peer Review Rubric: SQL Assignments

This rubric is designed to guide students in providing constructive, actionable feedback to their peers.

| Criteria | Exceptional (4) | Proficient (3) | Developing (2) | Needs Improvement (1) |
| :--- | :--- | :--- | :--- | :--- |
| **Code Readability** | Uses clear naming, proper indentation, and CTEs where appropriate. | Code is readable and clean. | Hard to follow; lacks structure. | Indecipherable. |
| **Logic/Efficiency** | Demonstrates optimal join usage and query structure. | Logical but potentially inefficient. | Logical errors present. | Fails to solve the problem. |
| **Edge Case Handling**| Proactively handles NULLs, empty sets, and data type issues. | Handles basic scenarios. | Basic scenarios are brittle. | Fails on edge cases. |
| **Feedback Quality** | Provides specific, constructive suggestions for improvement. | Provides general suggestions. | Minimal/generic feedback. | No feedback provided. |
